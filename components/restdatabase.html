This component can perform some operations on a database with the help of WhoMeta QueryBuilder. First, you must initialize the connection string for the database via QueryBuilder initialization. __Input data__: - `data {Object}` or `body {Object}` optional - payload - `query {Object}` optional - query data, key/value must be string - `params {Object}` optional - dynamic values, key/value must be string - `user {Object}` optional - a user instance __Output__: - `insert` returns Number - `update` returns Number - `remove` returns Number - `query` returns Array of Objects - `read` returns Object - `list` returns: ```json { items: Array, page: Number, pages: Number, count: Number } ``` Query parameters for the `List` operation can contain the below fields which can affect filtering according to the values defined in the dynamic filter. - `sort {String}` can contain field name with `_asc` or `_desc` value - `fields {String}` can return only specified fields separated by comma - `page {Number}` page number - `limit {Number}` max. items per page (default: max. limit) - `column_name=VALUE` enables dynamic filtering --- __Good to know__: If the `data` (payload) can contain special key names in the form: - `+key` increments a value - `-key` decrements a value - `!key` performs toggle for boolean values - `>key` stores only greater value - `
