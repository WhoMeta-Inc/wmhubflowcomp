This component can help you with generating confirmation codes. Keep data structure for inputs according to the documentation below. ### `create` input/output __Create input__: - `id {String}` identifier (email, phone number, identifier, etc.) - `param {String}` a custom parameter __Create output__: - `success {Boolean}` determines if the code is verified or not accroding to its identifier - `value {String}` a pin/code/token - `param {String}` a custom parameter (it sends only if the value is true) - `error {String}` (optional) a simple error identifier, it can contain: - `locked` the identifier is locked by the previous call ### `verify` input/output __Verify input__: - `id {String}` identifier (email, phone number, identifier, etc.) - `value {String}` a pin/code/token __Verify output__: - `success {Boolean}` determines if the code is verified or not accroding to its identifier - `slots {Number}` free slots/attempts - `error {String}` (optional) a simple error identifier, it can contain: - `invalid` the code is invalid - `404` identifier not found --- ### Settings - __Lock__ prevents overwriting of the code according to the identifier
